# Institutional Adoption FAQ

- methodology: `long-term-integrated-thesis`
- status: candidate_internal_release
- prepared_date: 2026-05-30
- stale_after: 2026-06-30
- must_refresh_if: reviewer findings, true follow-through refresh, release gate status, or validation case evidence changes

## Is This Ready For Institutional Colleagues?

Not as a final external-release methodology.

It is ready for internal candidate use and external reviewer assignment. Final release still requires `G04` true follow-through refresh and `G06` external independent review.

## What Counts As Institutional Acceptance?

Packet existence does not count.

Before final colleague distribution, at least one institutional colleague must use the packet without live author context, reproduce one existing case label, start one new case or approved refresh path, confirm source-gap visibility, challenge methodology-iteration traceability and sign a dated acceptance memo. Track this in `institutional-colleague-acceptance-checklist.csv`, validate the memo with `scripts/validate_institutional_colleague_acceptance_return.py`, and validate final state with `scripts/validate_institutional_colleague_acceptance.py`.

## What Problem Does It Solve?

It prevents long-term research from becoming narrative accumulation. The workflow forces a thesis to survive six linked lenses and a valuation expectation map before actionability is claimed.

## Is This A Stock-Picking Shortcut?

No.

This is not a stock-picking shortcut.

The workflow often blocks stock picking. For example, it can route a hot theme to `industry_map_first` when public-company value capture is unclear.

## How Is It Different From Ordinary Research?

Ordinary research can produce a coherent story. This workflow asks whether the story changes an investable decision after source gaps, stop rules and expectations are made explicit.

The current validation set shows actionability changed or was downgraded across ETN, VRT, CRM, LLY, TDOC, PTON and humanoid robotics.

## What Should A PM Trust?

Trust the discipline more than any single case output.

Useful outputs:

- weakest-lens identification
- stop rule
- expectation map
- source gap list
- refresh trigger
- ordinary-vs-workflow delta

Do not trust a conclusion when the evidence log or expectation map is incomplete.

## What Is Still Unproven?

- Whether the refresh loop works after a later real-world event.
- Whether an independent reviewer can reproduce and challenge conclusions without hidden context.
- Whether historical failure cases can be made fully transcript-backed and consensus-backed.

## What Must A Reviewer Challenge?

At minimum:

- whether the action label follows from the evidence log
- whether source gaps are visible enough
- whether the weakest lens is correctly identified
- whether the expectation map is sufficient
- whether MarketScreener FY2028 FCF is acceptable for CRM G05
- whether the workflow produces value beyond ordinary Mira research

## What Should Never Be Removed From A Shared Pack?

- current release status
- G04 and G06 caveats
- source gaps
- institutional acceptance status
- stop rules
- stale-after and refresh conditions
- reviewer findings or unresolved P1 caveats
